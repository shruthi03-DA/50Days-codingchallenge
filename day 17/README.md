🧾 Project Explanation: Customer Orders & Payments Analysis
📌 1. Project Overview
This project is about analyzing customer purchases and payment status using a relational database.

👉 We created a system with 3 main tables:
Customers → Stores customer details
Orders → Stores order transactions
Payments → Stores payment status

🎯 Goal:
To understand:
Who placed orders
Who hasn’t placed orders
Payment completion status
Customer spending behavior

🎯 Key Insights (Conclusion)
Technical Skills Demonstrated:
Joins (INNER, LEFT)
Aggregations (SUM, MAX)
Filtering (WHERE, HAVING)
Data validation logic
👉 “We built a relational database system to track customers, their orders, and payments. 
Using SQL queries, we analyzed customer activity, identified top spenders, detected unpaid orders, and ensured data accuracy. 
This helps businesses make better decisions and improve customer management.”
